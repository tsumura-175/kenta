# SOTO ASOBI PLAY & MOVE コーディング規約

このファイルは、LPを保守・追加修正するときの実装ルールです。

## HTML

- 意味のある要素を優先する。セクションは `section`、導線は `a`、操作は `button` を使い分ける。
- 見出しは `h1` をページに1つ、以降は順番に `h2`、`h3` を使う。
- クラス名は役割を表す英小文字のケバブケースにする。見た目だけの名前や略語は避ける。
- 共通CTAには `button`、用途に応じて `button--primary`、`button--compact`、`button--block`、`button--text` を追加する。
- 装飾画像には `alt="" aria-hidden="true"`、内容を伝える写真には具体的な代替テキストを付ける。

## CSS

- 定義順は「トークン・リセット → 基本要素 → レイアウト → 共通コンポーネント → セクション固有 → メディアクエリ」とする。
- 色、余白、角丸、影、文字サイズ、アニメーション時間は可能な限りカスタムプロパティを使う。
- 同じセレクタを後半で上書きしない。変更は元の定義へ戻して行う。
- セクション固有の指定は、そのセクションのブロック内に閉じる。
- `!important` は使わない。詳細度を上げて解決する前に、構造と共通トークンを見直す。
- レスポンシブ指定は `720px`、`1100px` ごとにまとめ、モバイルの定義を先に書く。

## JavaScript

- 機能単位で `init...` 関数に分ける。
- DOM要素がないページでも例外にならないよう、イベント登録前に存在を確認する。
- スクロール、アニメーション、フォーム、メニューの責務を混ぜない。
- 既存の `prefers-reduced-motion` を尊重し、動きのない代替表示を必ず用意する。
- 文言、時間、セレクタは役割ごとの定数にまとめる。

## 確認手順

1. HTMLの見出し順、フォームのラベル、リンク先を確認する。
2. 375px・768px・1440pxで横スクロールや意図しない折り返しがないか確認する。
3. キーボード操作でメニュー、FAQ、フォーム、CTAにフォーカスできるか確認する。
4. `prefers-reduced-motion` でもコンテンツが表示されるか確認する。
5. `git diff --check` とHTML構造チェックを実行する。
